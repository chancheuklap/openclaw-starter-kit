# 🦞 OpenClaw 多 Agent 实战配置模板

> 这是一份经过数周打磨的 OpenClaw 生产环境配置，包含三个 AI Agent 协同工作的完整架构。
> 你可以直接基于此模板修改，省去从零摸索的时间。

---

## 📋 快速上手

1. 备份你现有的 `~/.openclaw/openclaw.json`
2. 将 `openclaw.json` 复制到 `~/.openclaw/openclaw.json`
3. 搜索所有 `YOUR_` 开头的占位符，替换为你自己的密钥和 ID
4. 运行 `openclaw gateway restart` 生效

> ⚠️ **首次使用 OpenClaw？** 建议先跑一遍 `npx openclaw@latest` 的引导流程，再参考本配置修改。

---

## 🏗️ 架构总览

本配置实现了 **三 Agent + 双平台** 的架构：

```
┌─────────────────────────────────────────────────┐
│                   OpenClaw Gateway               │
├────────────┬────────────────┬───────────────────┤
│  main 🦞   │  coach 👨‍🏫     │  info 🔍          │
│  日常助理   │  专项顾问      │  信息采集          │
├────────────┴────────────────┴───────────────────┤
│         Telegram  ←  bindings  →  Discord        │
└─────────────────────────────────────────────────┘
```

每个 Agent 有独立的：
- **Workspace**（工作目录）— 文件互不干扰
- **Bot 账号**（Telegram + Discord 各一个）— 独立身份
- **Heartbeat**（心跳巡检）— 自主运行
- **人设文件**（SOUL.md / AGENTS.md）— 性格和职责各异

## 📖 配置详解

### 🔐 auth — 认证配置

```jsonc
"auth": {
  "profiles": {
    // OpenAI Codex — 用 OAuth 登录，不需要 API Key
    "openai-codex:default": { "provider": "openai-codex", "mode": "oauth" },
    // Anthropic Claude — 用 API Token（从 console.anthropic.com 获取）
    "anthropic:your_profile_name": { "provider": "anthropic", "mode": "token" },
    // Google Gemini — 用 OAuth 登录 Google 账号
    "google-gemini-cli:your_email@gmail.com": { "provider": "google-gemini-cli", "mode": "oauth" }
  }
}
```

**要点：** OpenClaw 支持多个 AI 提供商并存。OAuth 模式不需要手动管理 key，token 模式需要你自己从对应平台获取。

### 🤖 agents.defaults — 全局 Agent 默认值

```jsonc
"defaults": {
  // 主力模型：Claude Opus（最强推理能力）
  "model": { "primary": "anthropic/claude-opus-4-6" },

  // 可用模型列表 + 别名（方便在聊天中用 /model Sonnet 快速切换）
  "models": {
    "anthropic/claude-sonnet-4-6": { "alias": "Sonnet" },      // 性价比之选
    "anthropic/claude-opus-4-6": { "alias": "Opus" },           // 最强推理
    "google-gemini-cli/gemini-3-pro-preview": { "alias": "Gemini Pro" },  // Google 旗舰
    "google-gemini-cli/gemini-3-flash-preview": { "alias": "Gemini Flash" }, // 最便宜
    "openai-codex/gpt-5.3-codex": { "alias": "Codex" }         // 代码专用
  },

  // 向量记忆搜索 — 让 Agent 能搜索历史记忆文件
  "memorySearch": {
    "provider": "gemini",
    "remote": { "apiKey": "YOUR_GOOGLE_API_KEY" },  // Google AI API Key
    "model": "gemini-embedding-001"  // 嵌入模型，用于语义搜索
  },

  // 上下文管理 — 防止长对话爆 token
  "contextPruning": { "mode": "cache-ttl", "ttl": "1h" },  // 1小时前的消息逐步淡出
  "compaction": { "mode": "safeguard" },  // 触及上下文上限时自动压缩

  // 思考模式 — high = 深度推理（更慢但更准）
  "thinkingDefault": "high",

  // 消息分块输出 — 模拟人类打字，不再一坨丢出来
  "blockStreamingDefault": "on",
  "blockStreamingBreak": "text_end",
  "blockStreamingChunk": {
    "minChars": 80,   // 每块最少 80 字符
    "maxChars": 600,  // 每块最多 600 字符
    "breakPreference": "paragraph"  // 优先在段落边界断开
  },
  "blockStreamingCoalesce": {
    "minChars": 150,  // 太短的块合并
    "maxChars": 800,  // 合并后上限
    "idleMs": 300     // 300ms 内的连续块合并
  },

  // 模拟人类打字节奏
  "humanDelay": { "mode": "natural" },

  // 心跳巡检 — Agent 每小时自动 check-in
  "heartbeat": {
    "every": "1h",
    "model": "anthropic/claude-sonnet-4-6",  // 心跳用便宜模型，省钱
    "target": "discord",
    "to": "YOUR_DISCORD_USER_ID"  // 你的 Discord 用户 ID
  },

  // 并发控制
  "maxConcurrent": 4,          // 主 session 最多 4 个并发
  "subagents": { "maxConcurrent": 8 }  // 子 agent 最多 8 个并发
}
```

### 👥 agents.list — Agent 定义

```jsonc
"list": [
  {
    "id": "main",              // 默认 Agent，日常万能助理
    "identity": { "name": "my_claw", "emoji": "🦞" }
  },
  {
    "id": "coach_claw",        // 专项顾问 Agent
    "workspace": "~/.openclaw/workspace-coach_claw",  // 独立工作目录
    "agentDir": "~/.openclaw/agents/coach_claw/agent", // 人设文件目录
    "identity": { "name": "coach_claw", "emoji": "👨🏻‍🏫" }
  },
  {
    "id": "info_claw",         // 信息采集 Agent
    "workspace": "~/.openclaw/workspace-info_claw",
    "agentDir": "~/.openclaw/agents/info_claw/agent",
    "identity": { "name": "info_claw", "emoji": "🔍" }
  }
]
// 每个 Agent 都继承 defaults 里的 heartbeat 配置
// 你也可以在单个 Agent 里覆盖（比如不同的心跳频率）
```

**为什么要多 Agent？** 不同 Agent 有不同的人设和记忆。比如 coach 专注某个领域的知识积累，info 专注信息收集，main 处理日常。各自的 SOUL.md 和 MEMORY.md 互不干扰。

### 🔗 bindings — 消息路由

```jsonc
"bindings": [
  // Telegram: 每个 bot 对应一个 Agent
  { "agentId": "main",       "match": { "channel": "telegram", "accountId": "my_claw" } },
  { "agentId": "coach_claw", "match": { "channel": "telegram", "accountId": "coach_claw" } },
  { "agentId": "info_claw",  "match": { "channel": "telegram", "accountId": "info_claw" } },
  // Discord: 同理
  { "agentId": "main",       "match": { "channel": "discord", "accountId": "my_claw" } },
  { "agentId": "coach_claw", "match": { "channel": "discord", "accountId": "coach_claw" } },
  { "agentId": "info_claw",  "match": { "channel": "discord", "accountId": "info_claw" } }
]
// 核心逻辑：消息从哪个 bot 进来 → 路由到对应的 Agent 处理
// 一个 Agent 可以绑定多个平台（Telegram + Discord 同时响应）
```

### 📡 channels.telegram — Telegram 配置

```jsonc
"telegram": {
  "enabled": true,
  "dmPolicy": "allowlist",         // 只接受白名单用户的私聊
  "allowFrom": ["YOUR_TELEGRAM_USER_ID"],  // 你的 Telegram 数字 ID
  "groupPolicy": "allowlist",      // 群聊也需要白名单
  "streaming": true,               // 启用流式输出（打字机效果）

  "accounts": {
    "my_claw": {
      "botToken": "YOUR_TELEGRAM_BOT_TOKEN_1",  // @BotFather 创建的 bot token
      "groups": {
        "-100XXXXXXXXXX": {       // 群聊 ID（负数开头）
          "requireMention": false, // 不需要 @提及就响应
          "groupPolicy": "open"
        }
      },
      "ackReaction": "🦞"         // 收到消息时的确认表情
    }
    // coach_claw 和 info_claw 同理，各自有独立 bot
  }
}
```

**安全提醒：** `dmPolicy: "allowlist"` + `allowFrom` 是必须的！否则任何人都能跟你的 bot 对话，消耗你的 API 额度。

### 💬 channels.discord — Discord 配置

```jsonc
"discord": {
  "enabled": true,
  "token": "YOUR_DISCORD_BOT_TOKEN_DEFAULT",  // 默认 bot token
  "groupPolicy": "allowlist",
  "textChunkLimit": 1800,          // Discord 消息上限 2000 字，留 200 缓冲
  "mediaMaxMb": 50,                // 文件上传上限 50MB

  // Markdown 适配 — Discord 不支持表格，自动转成列表
  "markdown": { "tables": "bullets" },

  // 消息输出模式 — 关闭 streaming，用 blockStreaming 替代
  "blockStreaming": false,
  "streamMode": "off",
  "draftChunk": {                  // 分块输出参数
    "minChars": 200, "maxChars": 1800, "breakPreference": "paragraph"
  },

  // Discord 特有能力
  "actions": {
    "roles": true,       // 允许管理角色
    "moderation": true,  // 允许踢人/禁言
    "presence": true     // 允许设置在线状态
  },

  // 安全：命令执行需要审批
  "execApprovals": {
    "enabled": true,
    "approvers": ["YOUR_DISCORD_USER_ID"],  // 只有你能批准
    "target": "channel"  // 审批消息发到当前频道
  },

  "accounts": {
    "my_claw": {
      "token": "YOUR_DISCORD_BOT_TOKEN_1",
      "guilds": {
        "YOUR_GUILD_ID_1": {
          "requireMention": true,  // 大群里需要 @提及
          "channels": {
            "YOUR_CHANNEL_ID_1": {
              "allow": true,
              "requireMention": false  // 专属频道不需要 @
            }
          }
        },
        "YOUR_GUILD_ID_2": {
          "requireMention": false  // 私人服务器不需要 @
        }
      },
      "voice": {
        "enabled": false,  // 语音功能（需要 TTS 配置）
        "tts": { "provider": "openai", "openai": { "voice": "alloy" } }
      }
    }
    // coach_claw, info_claw 同理
  }
}
```

### 🛡️ gateway — 网关配置

```jsonc
"gateway": {
  "port": 18789,          // 本地监听端口
  "mode": "local",        // 本地模式（也支持 tailscale 远程）
  "bind": "loopback",     // 只监听 127.0.0.1
  "auth": {
    "mode": "token",
    "token": "YOUR_GATEWAY_TOKEN"  // 运行 onboard 时自动生成
  },
  "nodes": {
    // 禁止远程节点执行的敏感操作
    "denyCommands": [
      "camera.snap", "camera.clip", "screen.record",
      "calendar.add", "contacts.add", "reminders.add"
    ]
  }
}
```

## 💡 使用建议

### 成本控制
- **心跳用 Sonnet**（比 Opus 便宜 5 倍），日常对话用 Opus
- **定时脚本用 Gemini Flash**（最便宜，甚至有免费额度）
- **本地 Whisper 处理语音**，不调用云端 API

### 安全加固
- 所有 `dmPolicy` 设为 `allowlist`，杜绝陌生人白嫖
- 开启 `execApprovals`，危险命令需要你手动批准
- Gateway 绑定 `loopback`，不暴露到公网

### 多 Agent 最佳实践
- 每个 Agent 给一个清晰的职责定位（写在 SOUL.md 里）
- 不要让所有 Agent 共享一个 workspace
- 心跳频率根据需要调整（不活跃的 Agent 可以设 4h 甚至关掉）

## 🔑 需要替换的占位符清单

| 占位符 | 说明 | 获取方式 |
|--------|------|----------|
| `YOUR_GOOGLE_API_KEY` | Google AI API Key | [aistudio.google.com](https://aistudio.google.com/apikey) |
| `YOUR_BRAVE_SEARCH_API_KEY` | Brave 搜索 API Key | [brave.com/search/api](https://brave.com/search/api/) |
| `YOUR_TELEGRAM_BOT_TOKEN_*` | Telegram Bot Token | Telegram 搜索 @BotFather，创建 bot |
| `YOUR_DISCORD_BOT_TOKEN_*` | Discord Bot Token | [discord.com/developers](https://discord.com/developers/applications) |
| `YOUR_GATEWAY_TOKEN` | Gateway 认证 Token | `openclaw onboard` 自动生成 |
| `YOUR_TELEGRAM_USER_ID` | 你的 Telegram 数字 ID | Telegram 搜索 @userinfobot |
| `YOUR_DISCORD_USER_ID` | 你的 Discord 用户 ID | Discord 设置 → 高级 → 开发者模式 → 右键自己头像 → 复制 ID |
| `YOUR_GUILD_ID_*` | Discord 服务器 ID | 右键服务器图标 → 复制 ID |
| `YOUR_CHANNEL_ID_*` | Discord 频道 ID | 右键频道名 → 复制 ID |
| `your_email@gmail.com` | Google 账号 | 你的 Gmail 地址 |
| `your_profile_name` | Anthropic profile 名 | 自定义，用于区分多个 key |
